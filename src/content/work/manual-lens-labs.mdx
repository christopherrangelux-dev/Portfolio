## The Problem

Manual mode intimidates beginners. Every existing simulator I found shows sliders and a number — but a number doesn't teach. The exposure triangle is fundamentally about trade-offs: widening your aperture to get background blur means you're also letting in more light, which may mean speeding up the shutter, which may mean freezing motion you wanted to keep.

You need to *see* those trade-offs cascade to understand them. A static calculator can't do that.

## Engineering Notes

**Two-layer scene composition.** The viewfinder splits background and subject into separate DOM layers (`.mll-bg-layer` and `.mll-subject-layer`) rather than rendering a single flat frame. This makes selective blur possible: a CSS `filter: blur()` on the background layer simulates shallow depth of field without touching the subject layer. If both lived in one element, bokeh would blur everything equally and the core teaching moment — wide aperture isolates the subject — would disappear. The browser compositor handles the two `position: absolute` layers with no layout cost.

**Canvas grain loop.** Film grain is tied to ISO: a `requestAnimationFrame` loop redraws pixel noise on a `<canvas>` overlay every frame. The canvas is transparent at ISO 100 and increasingly visible at ISO 3200+. Each frame generates a fresh `ImageData` array with randomized luminance offsets, then writes it to the canvas with `putImageData`. The overlay uses `mix-blend-mode: overlay` so the grain interacts with the scene colors rather than just sitting on top.

```typescript
function drawGrain(canvas: HTMLCanvasElement, intensity: number) {
  const ctx = canvas.getContext('2d');
  if (!ctx) return;
  const imageData = ctx.createImageData(canvas.width, canvas.height);
  for (let i = 0; i < imageData.data.length; i += 4) {
    const noise = (Math.random() - 0.5) * intensity * 80;
    imageData.data[i] = imageData.data[i+1] = imageData.data[i+2] = 128 + noise;
    imageData.data[i+3] = intensity * 60;
  }
  ctx.putImageData(imageData, 0, 0);
}
```

The loop cancels cleanly on Astro View Transitions via `astro:before-swap`, preventing orphaned `requestAnimationFrame` callbacks from running after navigation:

```typescript
document.addEventListener('astro:before-swap', () => cancelAnimationFrame(rafId));
```

**Scene matrix.** `assignScene()` evaluates the three sliders in priority order, first match wins. This keeps the logic flat and auditable — no nested state machine, just a ranked list of conditions:

| Priority | Condition | Scene | Subject |
|---|---|---|---|
| 1 | ISO ≥ 3200 and shutter ≥ 1/30s | Night Street — Moving Traffic | Runner |
| 2 | ISO ≥ 3200 | Night Street | Person |
| 3 | Shutter ≥ 1/30s and ISO < 3200 | Runner Mid-Stride | Runner |
| 4 | Shutter ≤ 1/1000s and aperture ≤ f/2.8 | Wildlife — Bird in Flight | Bird |
| 5 | Aperture ≤ f/1.8, ISO ≤ 400, moderate shutter | Golden Hour Portrait | Person |
| 6 | Aperture ≥ f/8 and ISO ≤ 400 | Landscape — Mountain Vista | Landscape |
| 7 | EV ≤ −2 stops from neutral | Dark Interior | Person |
| 8 | EV ≥ +2 stops from neutral | Blown-Out Beach | Person |
| 9 | (default) | Street Scene, Daylight | Person |

**Critique engine.** The critique is rule-based and fully deterministic: same settings, same output, every time. The rules are ordered by severity (severe underexposure before subtle underexposure before motion blur before grain before DOF notes). This architecture is intentionally AI-upgradeable — the rule table maps directly to a system prompt, and an LLM layer would plug in at the same function interface (`generateCritique(): string`) with no changes to the rest of the component.

## Reflection

- User-uploadable background photos — replace the CSS gradient scenes with real images
- Histogram display — luminance distribution of the simulated exposure plotted live
- Save and compare two shots side by side
- Challenge mode — match a target exposure for a given scene
- AI critique upgrade — plug directly into the existing `generateCritique` interface with no architecture changes required
- White balance control — the notable v1 omission; color temperature is the fourth variable the exposure triangle ignores
